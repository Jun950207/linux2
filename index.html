第五章 權限管理與程序
5.1 傳統權限 rwx

ls -l 輸出格式（期中考考過，期末很可能再考）

-rwsr-xr-x 1 root root 32 2月26 09:10 /usr/bin/passwd

欄位說明範例
1檔案類型 + 權限-rwsr-xr-x
2連結數1
3擁有者（owner）root
4擁有群組（group）root
5檔案大小32
6修改時間2月26 09:10
7檔名/usr/bin/passwd

chmod 數字對應

r
= 4（讀）
w
= 2（寫）
x
= 1（執行）
rwx
= 7
r-x
= 5
r--
= 4
# 設定擁有者rwx 群組r-x 其他r-x
chmod 755 /path/to/file

# 設定擁有者與群組
chown Jack:Peter myGod
5.2 特殊權限 SUID / SGID / SBIT ⭐最常考
名稱設在哪效果ls 顯示數字
SUID 指令（檔案） 執行時暫時取得擁有者的權限 擁有者 x → s 4xxx
SGID 指令 or 目錄 在目錄下建立的新檔案，群組自動繼承目錄群組 群組 x → s 2xxx
SBIT 目錄 目錄內的檔案，使用者只能刪自己建立的 其他 x → t 1xxx
📌 記憶口訣： SUID=4、SGID=2、SBIT=1，跟 rwx 的 4-2-1 一樣！
範例： chmod 6755 = SUID(4)+SGID(2), rwxr-xr-x 
# 設定 SUID（4+755）
chmod 4755 /usr/local/bin/mycat2

# 設定 SUID + SGID（4+2+755 = 6755）
chmod 6755 /usr/bin/passwd

# 設定 SBIT（1+777 = 1777，如 /tmp）
chmod 1777 /tmp

# 確認 SBIT 是否存在（看 t 字元）
ls -ld /tmp
⚠️ 重要： passwd 能修改 /etc/shadow 就是因為 passwd 有 SUID，執行時暫時取得 root 權限。這個例子考試很愛出！
5.3 程序管理 (Process)

NI 值（Nice Value）：決定優先權，範圍 -20 到 19

NI 值越小 → 優先權越高 → 越早執行

NI 值越大 → 優先權越低 → 越晚執行

# 查看程序（含 PID, PRI, NI, COMMAND）
ps -eo pid,pri,ni,comm | head

# 找特定程序
ps aux | grep crond

# 修改執行中程序的 NI 值
renice -n -15 [PID號碼]

# 以指定 NI 值啟動程式
nice -n 10 su student

# 即時監控（每2秒更新）
top -d 2
# top 內按 P = 依 CPU 排序；按 M = 依記憶體排序
5.4 工作管理 (Job Control)
操作指令/按鍵說明
丟到背景（暫停）Ctrl + Z程序暫停，放入背景
丟到背景（執行）指令 &直接在背景執行
查看背景工作jobs -l顯示工作號碼與 PID
移到前景fg %1%1 = 工作號碼 1
在背景繼續執行bg %1讓暫停的工作在背景跑
中斷程序Ctrl + C強制終止
結束輸入（EOF）Ctrl + D不是中斷！是輸入結束
📌 Ctrl+Z vs Ctrl+C vs Ctrl+D：
Z = 暫停放背景 ｜ C = 直接殺掉 ｜ D = 結束輸入（cat 用這個）
第六章 磁碟分割與檔案系統
6.1 磁碟分割（MBR vs GPT）
MBRGPT
最大磁碟2 TB無限制（實際 9.4 ZB）
主分割數最多 4 個128 個（不分主/邏輯）
延伸分割需要（突破 4 個限制）不需要
何時使用磁碟 ≤ 2TB磁碟 > 2TB

MBR 裝置檔名規則（裝置為 /dev/sda 為例）

主分割：/dev/sda1, /dev/sda2, /dev/sda3

延伸分割：/dev/sda4（最多一個）

邏輯分割：/dev/sda5, /dev/sda6, /dev/sda7… 永遠從 5 開始

# 查看磁碟裝置
lsblk
lsblk -p   # 顯示完整路徑

# 查看分割表類型（MBR/GPT）
parted /dev/sda print

# 進入 fdisk 分割磁碟
fdisk /dev/sda
# 按 p = 查看, n = 新增, w = 儲存, q = 離開

6.3 inode、Block 與連結

EXT2 檔案系統三大區塊

① superblock：整個檔案系統的資訊（block/inode 總數等）

② inode table：儲存檔案的屬性、權限、時間等 metadata

③ block area：儲存實際檔案內容（程式碼、資料）

實體連結 (Hard Link)符號連結 (Symbolic Link)
建立指令ln 來源 目標ln -s 來源 目標
inode⭐ 相同 inode不同 inode
來源刪除後連結仍可使用連結失效（紅色顯示）
跨磁碟❌ 不可✅ 可以
類比兩個名字指同一資料捷徑/快捷方式
6.4 掛載與卸載
📌 掛載點是目錄，不是檔案！
# 建立掛載目錄並掛載
mkdir /srv/linux /srv/win
mount /dev/sda4 /srv/linux
mount /dev/sda5 /srv/win

# 查看目前掛載狀態（-T 顯示檔案系統類型）
df -T

# 卸載

第七章 Shell 變數與環境設定
7.1 變數設定語法 ⭐必考
語法用途範例
${ 變數 }引用變數內容echo ${PATH}
$( 指令 )執行指令並取得輸出kver=$(uname -r)
$(( 數學 ))數學計算result=$((3+5))
"雙引號"保留 $ 功能（變數會展開）"I am $myname"
'單引號'純文字（$ 不展開）'$myname' → 顯示 $myname
反引號 ` `同 $( )，舊式寫法`uname -r`
# 設定含空格的變數（必須加引號）
myname="peter pan"
echo $myname

# 動態取得核心版本
kver="my kernel version is $(uname -r)"

# 數學計算
result=$((100 + 200))

# 錯誤示範：變數名不能以數字開頭
# 2myname="test"  ← 這樣會報錯！
⚠️ 變數名稱不能以數字開頭，且 = 號兩側不能有空格！
7.2 區域變數 vs 全域變數

區域變數：只有目前 shell 可以使用，子程序看不到

全域變數（環境變數）：子程序也能繼承使用，用 export 設定

# 設定區域變數
mypp="hello"

# 轉為全域變數（子程序也能用）
export mypp

# 查看全域變數
env
export

# 查看所有變數（含區域）
set

# 常用環境變數
echo $PATH       # 執行路徑
echo $HISTSIZE   # 歷史命令筆數
echo $LANG       # 語系
echo $HOME       # 家目錄
7.3 Shell 設定檔 ⭐常考
情境讀取的設定檔
login shell（用帳密登入）~/.bash_profile 或 ~/.profile
non-login shell（在 bash 內再開 bash）~/.bashrc
所有使用者的系統設定/etc/profile
登出時執行~/.bash_logout
📌 讓設定立即生效（不登出）：
source ~/.bashrc 或 . ~/.bashrc

.bashrc 常見設定內容（考試愛考！）

# 設定 history 筆數
HISTSIZE=10000
export HISTSIZE=5000   # 設為全域

# 命令別名（alias）
alias cp='cp -i'    # cp 自動詢問是否覆蓋
alias rm='rm -i'
alias mv='mv -i'

# 設定語系（全域）
export LANG=zh_TW.utf8

# 擴充 PATH
PATH="${PATH}:~/cmd"
export PATH

# 提示字元（PS1）加入時間與指令次數
PS1='[\u@\h \t \# \W]\$ '
第八章 Script、資料流與管線
8.1 資料流重導向
符號功能範例
>stdout 輸出到檔案（覆蓋）ls > out.txt
>>stdout 輸出到檔案（累加）ls >> out.txt
2>stderr 輸出到檔案find 2> /dev/null
2>>stderr 累加到檔案
&>stdout + stderr 都導向cmd &> /dev/null
<stdin 從檔案讀入cat < /etc/hosts
/dev/null垃圾桶（丟棄輸出）2> /dev/null
# 找 passwd 相關檔案：丟棄錯誤訊息，正確輸出存檔
find /etc -name '*passwd*' 2> /dev/null > ~/find_passwd.txt

# 累加 shadow 相關結果
find /etc -name '*shadow*' 2> /dev/null >> ~/find_passwd.txt
8.2 管線命令 (pipe) 與邏輯運算子
符號說明
cmd1 | cmd2cmd1 的 stdout 傳給 cmd2 當 stdin
cmd1 && cmd2cmd1 成功（回傳0）才執行 cmd2
cmd1 || cmd2cmd1 失敗才執行 cmd2
echo $?查看上一個指令的回傳值（0=成功）
# 常用管線組合

# 統計 /etc 下 .conf 檔案數
find /etc | grep '\.conf' | wc -l

# 查 /etc/passwd 第1和第7欄（帳號+shell）
cat /etc/passwd | cut -d ':' -f 1,7

# 統計每個 shell 有幾個帳號
cat /etc/passwd | cut -d ':' -f 7 | sort | uniq -c

# 同時顯示螢幕又存檔（tee）
ip addr show | grep 'inet' | awk '{print $2}' | tee /dev/shm/myip.txt

# 條件判斷範例
# 不存在 /dev/shm/check 才建立
ls -ld /dev/shm/check || mkdir /dev/shm/check
8.3 Script 基礎與 test 判斷
#!/bin/bash
# test 判斷（常用參數）
test -e /etc/       # 是否存在
test -f /etc/hosts  # 是否為一般檔案
test -d /etc/       # 是否為目錄
test -u /usr/bin/passwd  # 是否有 SUID
echo $?             # 0=true, 1=false

# 用 [] 代替 test（注意空格！）
[ -e /etc/ ]; echo ${?}
[ ${HOSTNAME} == "mylocalhost" ]; echo $?

# 簡單 if 判斷（用 && ||）
test -e ${1} && echo "exist" || echo "non-exist"

() 的用途：將多個指令合併，統一重導向

date; uptime; uname -r > myfile.txt → 只有 uname -r 的輸出進 myfile

(date; uptime; uname -r) > myfile.txt → 三個指令都進 myfile
